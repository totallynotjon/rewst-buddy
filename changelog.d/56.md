- **MCP server (experimental): workflow tools** — external MCP clients (Claude Desktop, Claude Code, Cursor) can now read, debug, and edit Rewst workflows through the same helpers the Cage-Free Rewsty chat uses, when workflow tools are enabled. Six read-only tools: `buddy_workflow_get` (read a workflow as a node/edge graph), `buddy_workflow_search` (find workflows by name across your orgs), `buddy_action_search` (find or describe actions), `buddy_workflow_executions` (list recent runs), `buddy_execution_logs` (per-task logs for one run), and `buddy_render_jinja` (evaluate a Jinja expression against an execution). Three workflow write tools — `buddy_workflow_edit`, `buddy_workflow_autolayout`, and `buddy_workflow_run` — are also exposed when write tools are enabled, and each requires a per-edit approval inside VS Code before it runs.
