- **Workflow action edits no longer corrupt or drop a task's settings** — editing a task through `buddy_workflow_edit` now round-trips every advanced task setting (integration/pack override, retry, timeout, run-as org, mock, concurrency/loop `with`, and the rest) instead of silently reverting it to a default, and structured fields supplied as JSON-encoded strings (`input`, `with`) are parsed back into objects instead of being stored as character-indexed blobs. Numeric settings (`join`, `timeout`) given as strings are coerced, and non-object/non-numeric values are rejected with a clear error rather than corrupting the task. The workflow graph view also surfaces a task's integration override so an edit visibly keeps it.
