- **MCP server (foundation, experimental)** — Rewst Buddy can now expose your authenticated Rewst sessions to external MCP clients (Claude Desktop, Claude Code, Cursor) over a local HTTP endpoint served from the extension itself — no separate process and no `node` required. Clients connect to a `http://<host>:<port>/mcp` URL (default `http://127.0.0.1:27121/mcp`) and present a per-install localhost token in the standard `Authorization: Bearer` header; the generated config stays credential-free by referencing the token via the `REWST_BUDDY_MCP_TOKEN` environment variable, and your Rewst cookies never leave the extension. Off by default behind `rewst-buddy.mcp.enable`; read-only at the boundary (writes are rejected unless `rewst-buddy.mcp.enableWriteTools` is set), with an optional `rewst-buddy.mcp.enabledTools` allowlist. For VS Code's own MCP client, the new **Add MCP Server to VS Code** command registers the server natively (it shows up in the editor's MCP server list with the localhost token wired in — nothing to copy); for external clients (Claude Desktop, Claude Code, Cursor), the **Copy MCP Config to Clipboard** command produces the credential-free JSON config. This release lands the plumbing and a capability registry shared with the Cage-Free Rewsty chat tools; the GraphQL introspect/query/mutate tools are wired onto the MCP surface in a follow-up.
